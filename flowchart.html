Dynamic Quantization — Layer-by-Layer Flow

How activations are quantized, multiplied, dequantized, and passed forward

Input Activation  X FP32 shape: [batch, in_features]
LAYER i  (e.g. nn.Linear)
1
Scan Activation Range FP32
Walk through every element of X to find the min and max values. This full-tensor scan is the runtime cost of "dynamic" quantization.
x_min = min(X)    // e.g. 0.0
x_max = max(X)    // e.g. 6.0
2
Compute Scale & Zero Point
Map the float range onto INT8  [−128, 127]. The scale is the step-size per integer tick. The zero point ensures 0.0 maps to an exact integer.
sx = (x_max − x_min) / (127 − (−128))
    = (x_max − x_min) / 255

zx = round(−128 − x_min / sx)
    // clamped to [−128, 127]
3
Quantize Activations FP32 → INT8
Every element is scaled, shifted, rounded, and clamped to produce an INT8 tensor Xq of the same shape.
Xq[i] = clamp( round( X[i] / sx + zx ), −128, 127 )

// Example: X[i] = 3.5, sx = 0.02353, zx = −128
// Xq[i] = round(3.5 / 0.02353 + (−128)) = round(20.7) = 21
INT8 Matrix Multiplication  INT8 × INT8 → INT32

We have two INT8 tensors ready:

Activations (dynamic)
Xq  INT8
scale: sx   zero: zx
quantized just now ↑
Weights (pre-quantized)
Wq  INT8
scale: sw   zero: zw
stored offline at load time

The real-valued output Y = X · WT is decomposed as:

// Real relationship:
X ≈ sx · (Xq − zx)  W ≈ sw · (Wq − zw)

// Substituting into Y = X · Wᵀ :
Y = sx · sw · (Xq − zx) · (Wq − zw)T
    ↑ float scalar    ↑ integer-only matmul

// Hardware executes:
Yint32 = (Xq − zx) · (Wq − zw)T
// INT8 × INT8 → accumulated in INT32 registers
// (avoids overflow: N × 127 × 127 can exceed INT8)

This is where the 2–4× speedup lives: INT8 multiply-accumulate ops are much faster than FP32, and loading INT8 weights uses 4× less memory bandwidth.

5
Dequantize Result INT32 → FP32
Multiply the integer result by the combined scale to recover a floating-point output.
Yfloat = sx · sw · Yint32
// one scalar × entire tensor → back to FP32
6
Post-ops (all in FP32)
Add bias, apply activation functions (ReLU, GELU …), batch norm, residual connections, layer norm — all in floating point.
Yout = Activation( Yfloat + bias )
// e.g. ReLU( Yfloat + bias )
Yout travels as FP32 — quantization is gone, no INT8 data leaves the layer
LAYER i + 1
1
Scan New Activation Range FP32
The incoming tensor has completely different values now (post-ReLU, post-LayerNorm, etc.) so a fresh scan is needed.
x_min′ = min(Yout)   // new values!
x_max′ = max(Yout)   // different from layer i
2
Compute New Scale & Zero Point
Entirely new parameters — no relationship to Layer i's values.
sx′ = (x_max′ − x_min′) / 255
zx′ = round(−128 − x_min′ / sx′)
3
Quantize FP32 → INT8
Xq′[i] = clamp( round( Yout[i] / sx′ + zx′ ), −128, 127 )
INT8 Matmul (Layer i+1 Weights) INT8 × INT8 → INT32
Same principle, but with Layer i+1's pre-quantized weights (Wq′, sw′, zw′).
Yint32′ = (Xq′ − zx′) · (Wq′ − zw′)T

Yfloat′ = sx′ · sw′ · Yint32′ // dequantize
6
Post-ops → FP32 output
Bias + activation + normalization in FP32, then pass to the next layer.
Continues to Layer i+2 … FP32 same cycle repeats
Key Takeaway

Inside each layer the data lives as FP32 → INT8 → INT32 → FP32. Between layers it is always FP32. Quantization is transient — it exists only for the heavy matrix multiply where INT8 hardware delivers the speed and memory benefits.
